JavaScript Concepts Overview

1. Function vs Arrow Function

// Function Declaration
function add(a, b) {
  return a + b;
}

// Arrow Function
const addArrow = (a, b) => a + b;

Comparison Table

Feature Function Declaration Arrow Function
Syntax Verbose Concise
this binding Dynamic Lexical (inherits)
Usable as constructor ✅ Yes ❌ No
arguments object ✅ Yes ❌ No
Suitable for methods ✅ Yes ⚠️ Not recommended

2. Advantages of Arrow Functions

  • ✅ Lexical this binding (avoids rebinding)
  • ✅ Concise syntax for small functions
  • ✅ Works well with array methods: map, filter, reduce
  • ✅ Cleaner callback syntax in promises or event handlers

3. Disadvantages of Arrow Functions

  • ❌ Cannot be used as constructors with new
  • ❌ No access to arguments object
  • ❌ Not suitable as object methods if you rely on this
  • ❌ Cannot use super, new.target

5. arguments vs ...rest

Traditional arguments

function showArgs() {
  console.log(arguments); // array-like object
}

Modern ...rest (Recommended)

const showArgs = (...args) => {
  console.log(args); // real array
};

Comparison

Feature arguments ...rest
Works in arrow functions ❌ ✅
Real array ❌ ✅
Modern syntax ❌ ✅

6. All Uses of ... (Spread / Rest)

1. Rest Parameters (function definition)

function total(...nums) {
  return nums.reduce((a, b) => a + b, 0);
}

2. Spread in Function Calls

const nums = [1, 2, 3];
console.log(Math.max(...nums));

3. Spread in Arrays

const arr1 = [1, 2];
const arr2 = [3, 4];
const combined = [...arr1, ...arr2];

4. Spread in Objects

const user = { name: "Alice" };
const updated = { ...user, age: 25 };

5. Rest in Object Destructuring

const { name, ...rest } = { name: "Bob", age: 30, city: "NY" };
console.log(rest); // { age: 30, city: "NY" }

6. Rest in Array Destructuring

const [first, ...others] = [10, 20, 30];
console.log(others); // [20, 30]
