1. Geolocation API

üìç The Geolocation API lets a website access the user's current location (latitude & longitude). Commonly used in maps, weather apps, and delivery tracking.

Code Snippet:

if (navigator.geolocation) {
  navigator.geolocation.getCurrentPosition((position) => {
    const lat = position.coords.latitude;
    const long = position.coords.longitude;
    console.log(lat, long);
  }, () => {
    console.log("Unable to retrieve location.");
  });
} else {
  console.log("Geolocation is not supported.");
}

Explanation:

  • navigator.geolocation ‚Äì Checks if geolocation is supported.
  • getCurrentPosition() ‚Äì Asks the user for permission and gets the location.
  • position.coords.latitude / longitude ‚Äì Gives you the exact coordinates.
  • callback functions ‚Äì First is success, second is error handler.

2. Drag and Drop API

üß≤ The Drag and Drop API allows users to drag elements and drop them into designated drop zones. Useful in games, file uploads, sorting apps, etc.

Code Snippet:

// Step 1: Start dragging
dragItem.addEventListener("dragstart", (e) => {
  e.dataTransfer.setData("text/plain", "dropped");
});

// Step 2: Allow drop
dropZone.addEventListener("dragover", (e) => {
  e.preventDefault(); // Needed to allow drop
});

// Step 3: Handle drop
dropZone.addEventListener("drop", (e) => {
  e.preventDefault();
  dropZone.innerHTML = "‚úÖ Dropped Successfully!";
});

Explanation:

  • draggable="true" ‚Äì Makes the element draggable.
  • dragstart ‚Äì Triggered when dragging starts. You can attach data to the drag.
  • dragover ‚Äì Required to allow dropping by calling preventDefault().
  • drop ‚Äì Handles what happens when the draggable item is dropped.
